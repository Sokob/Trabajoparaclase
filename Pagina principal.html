HTML

¿Que es HTML?

HTML (HyperText Markup Language) hace referencia a los enlaces que conectan a páginas web entre sí, con una estructura basada en “marcas” para mostrar en pantalla los textos u otro contenido de índole multimedia en los navegadores. HTML cuenta con una estructura de inicio (head), un contenido (p, title, body, etc.), y un cierre (footer) para el desarrollo de los recursos en la programación web.

Principales etiquetas en HTML:

  • BODY: Indica la parte y contenido del cuerpo del documento HTML en el que se está trabajando.
  • HEAD: En esta sección podemos indicar los metadatos: título de la página, hojas de estilo, CSS, lenguaje.
  • DIV: La etiqueta div nos ayuda a mantener un control de las secciones con las que estamos trabajando y a buscarlos de forma más adecuada en el CSS.
  • A: Esta etiqueta nos ayuda a crear enlaces web o de referencia entre archivos, usado principalmente para indicar la existencia de contenido extra.
  • STRONG: Etiqueta que tienda a dar mucho más énfasis a los títulos y textos.
  • BR: Nos sirve para hacer un salto de línea
  • H1…H6: Esta etiqueta hará énfasis al tamaño de títulos que hay, siendo H1 el mas grande, y el H6 el más chico. Después del H2 estos tienden a cumplir la función de subtítulos.
  • IMG: Usado para mostrar imágenes en los archivos HTML.
  • UL: Nos será útil para crear lista que no tienen un orden, además estas se subdividen en:
  • OL: Listas ordenadas.
  • LI: Lista de elementos (no confundir con listas sin ordenar).
  • P: Nos sirve para poder encerrar nuestros textos en párrafos, puede ser combinado con la etiqueta br para poder crear saltos.
  • SPAN: Con esta etiqueta podemos personalizar el estilo de solamente una parte del texto de nuestro HTML.

La ventaja de HTML es que es un estilo de “programación” libre y de código abierto que permite a cualquier persona crear una simple pagina web, pero suele usarse como un método de enseñanza y para principiantes, esto debido a que se a que HTML a sido actualizado con etiquetas que resultan de fácil comprensión para todos los usuarios y resume ciertas funciones que ayudan al CSS.

CSS

¿Que es CSS?

CSS (Cascading Style Sheets) o hojas de estilo cascada es un lenguaje que otorga estilos de color, fuentes, diseños o espaciado para documentos web, esto quiere decir que el código CSS es la parte que todos vemos, la parte con colores y diseños, mientras que el HTML es la estructura base de la página web.

Para poder agregar un estilo CSS a nuestros archivos HTML, hace falta conocer 3 formas en las que podemos trabajar:

Hoja de estilo incrustada:

Un CSS puede incrustarse dentro de el mismo documento de HTML con una etiqueta “style” en la cabecera del documento HTML. El problema de este estilo es que solo es aplicable para un solo documento HTML, porque causaría problemas si se usa en mas de uno, en este casi, si se piensa trabajar con mas de un documento, es preferible usar una hoja de estilo externo.

Hoja de estilo en linea:

El estilo en línea se utiliza en cualquier parte del documento HTML, especificando el elemento que se va a modificar en HTML utilizando la etiqueta “style” dentro del elemento en el que se quiere poner dicho diseño. A diferencia del estilo incrustado y externo, este solo modificara un aspecto en específico del documento HTML, lo cual es mas recomendable intentar no usarlo de manera practica y cotidiana en los proyectos (solo como boceteado digital o para saber cómo afecta en el documento).

Hoja de estilo externa:

En este estilo se trabaja en una hoja externa de CSS, donde vamos a comunicarnos con dichos archivos desde el HTML utilizando la etiqueta “link” desde la cabecera del documento, se especifica la ruta de dirección donde se localiza el CSS con los estilos y se trabaja por separado. Como dato adicional, el atributo type es opcional y sirve para especificar un tipo de medio – text/CSS

Sintaxis de la regla CSS:

jerarquia del CSS

CSS utiliza una jerarquía hojas de estilo, lo cual va a afectar en cómo se utilizan en los documentos, a continuación, te enseño cuál es su jerarquía:

  1. Estilo de hoja en línea
  2. Estilo de hoja externa e interna (referido a los que se colocan en el encabezado)
  3. Los que usa el navegador propio (solo lo hará si el navegador no es capaz de encontrarlos)
LINKS

Etiqueta para links

Los links funcionan como un enlace entre un medio de información a otro, ya sean imágenes, videos, textos, se pueden colocar y referencias dentro del HTML. Para que los links puedan funcionar tienen que hacer uso de la sintaxis:

Estados de actividad

Los navegadores tienen estados en los que los links se pueden mantener, ya sean que estén activos o ya visitados, el navegador le hará saber al usuario si ya ha visitado dicho link o si hay muchos mas por visitar. Los links pueden tener representaciones para dichos estados de actividad, por ejemplo:

  • Un enlace no visitado puede estar subrayado y en color azul.
  • Un enlace visitado va a estar subrayado y tendrá un color morado.
  • Un enlace activo estará subrayado y en color rojo al momento de dar click.

Atributos especiales para la etiquetas con links

Dentro de los atributos de “a” nos encontramos con 3 estilos que nos ayudaran a brindar información y como herramienta de uso:

Link de imagen:

Para poner un link dentro de una imagen, solo basta con poner la etiqueta “img” dentro de la etiqueta “a”.

Link con atributo "title":

Un link con atributo “title” siempre va a especificar como se llama susodicho elemento dentro del documento, normalmente este objeto se visualiza su pasamos el cursor sobre algún elemento dentro de la página HTML.

Link con atributo "target":

Un link de atributo target va a especificar como vamos a abrir el link o documento, por ejemplo:

  • _self: Abre el documento en misma ventana o pestaña en la que se hizo el click
  • _blank: Abre el documento en otra ventana o pestaña
  • _parent: Abrirá el documento y lo ajustará al marco primario de la pagina
  • _top: Abrirá el documento de forma y tamaño completo en la pagina
HTML5

HTML5 (Nuevos elementos)

La versión 5 de HTML presenta una serie de elementos útiles para la web actual donde los podemos clasificar en dos formas:



NUEVAS FUNCIONALIDADES: HTML5 cumple con nuevas funciones que permiten una mejor interacción con el usuario, ya sean soportes para video o sonido, lienzos para dibujar, entre varios.

WEB SEMANTICA: Estas nuevas etiquetas proponen simplificar, tanto el lenguaje como el método de trabajo indicando los bloques específicos en el código de la web, en lugar de darles representación alguna, sin embargo, cabe resaltar dos cosas, existen dos tipos de web semántica:

  • Elementos NO semánticos: Son aquellas etiquetas o elementos del código que no dicen nada sobre su contenido, ejemplo las etiquetas: div y span.
  • Elementos semánticos: Son aquellos que, si definen algún tipo de contenido en el código y que, si funciona para algo, ejemplo de algunas etiquetas son: form, table y article.


Algunas de las etiquetas semánticas que se actualizaron y añadieron, fueron:

  • ARTICLE: Define información que funciona independiente del documento
  • ASIDE: Define contenidos que casi no se relacionan y que funcionan fuera de este
  • ADDRES: Define una sección que contiene información de contacto
  • FIGCAPTION: Representa la leyenda de una figura.
  • FIGURE: Representa una figura ilustrada como parte del documento.
  • FOOTER: Define el pie de página.
  • HEADER: Define la cabecera de la página.
  • MAIN: Define el contenido principal e importante del documento.
  • NAV: Define una sección que contiene información y enlaces.
  • SECTION: Define una sección dentro del documento.

Antiguamente con HTML4, los desarrolladores utilizaban sus propios identificadores para poder trabajar y aunque suele ser algo útil, cuando se comparte el trabajo de desarrollo se van a suscitar muchos problemas, tanto de comunicación, como de entendimiento por ambos lados, es por eso que HTML5 incluyo elementos (Header, Footer, Nav, Section, Article), gracias a esta actualización en los códigos, los navegadores pueden utilizar de forma eficiente y rápida en empresas, aplicaciones y comunidades.

Nuevas funciones:

Los elementos nuevos sirven para integrar contenido multimedia para agregar nueva información para la web:

  • AUDIO: Inserta audio en la página.
  • VIDEO: Inserta clips o videos desde internet o desde un archivo enlazado.
  • EMBED: Integra plugins.
  • SOURCE: Permite especificar varias pistas de sonido o audio para los elementos de audio y video.

Diseño web

Diseño web centrado en el usuario

Yusef Hassan, Francisco J. Martin y Ghzala lazza proponen la aplicación de un marco tecnológico conocido como Diseño Centrado en el Usuario, donde el diseñador es donde deberá aplicar metodologías, técnicas y procedimientos ideados con el fin de saber si un sitio web cumple con las especificaciones de usabilidad web. Con esto se asume que el proceso y desarrollo deberá cumplir con las necesidades, características y objetivos planteados, esto implica que:

  • Involucrar a los usuarios en el proceso para poder conocerlos y que es lo que necesitan
  • Investigar como estos reaccionan a la experiencia y al diseño
  • Si cumple con los estándares de innovación para mejorar la experiencia del usuario

En el proceso de diseño, varias de las fases tienen procesos iterativos, es decir, son procesos que se repiten constantemente, como lo serian las fases de “diseño”, “prototipado” y “evaluación”.



A continuación, vamos a explicar algunas de las etapas o fases del diseño:

Planificación:

En esta fase el diseñador estará en una constante investigación con los usuarios, preguntándose siempre el que es factible usar en la página (contenido y diseños) para poder lograr sus objetivos. Se basa casi completamente en recoger, analizar y ordenar toda la información posible en base a la información del usuario, contexto y objetivos claros con el sitio web.

Benchmarking:

Se hace una investigación a aquellas páginas que nosotros estemos visitando con el fin de observar, analizar y conseguir los detalles para poder diseñar de manera competitiva, intuitiva y analítica el como los usuarios han aceptado dichas paginas; la manera de realizar el benchmarking es plantearse una serie de preguntas:

  1. ¿Cuáles son los objetivos del sitio?
  2. ¿Cuánto tiempo tardo en cargar?
  3. ¿Se descargó en un orden lógico?
  4. ¿Es un sitio web interesante?
  5. ¿Cómo son los gráficos?
  6. ¿Refleja el género de la empresa o compañía?
  7. ¿Es de fácil uso y es cómodo navegar en él?
  8. ¿El contenido está estructurado de manera lógica?
  9. ¿Hay demasiados o insuficientes niveles de información?
  10. ¿En conjunto el sitio es un éxito?

Diseño:

En esta etapa simplemente se trata en la toma de decisiones acerca de lo que se plantea diseñar en base a la investigación obtenida para poder resolver, tanto problemas de usabilidad de otras páginas o hacer innovaciones.

Modelado del usuario:

A partir de este paso, debemos resumir y organizar toda la investigación en población que hicimos con nuestros usuarios, sin embargo, cuando se tienen demasiados usuarios a veces es muy complicado desarrollar cada situación en detalle, por lo creamos un perfil de identidad para poder emplear y poner toda la información en una sola hoja de información, creando un “escenario” y una “situación” que sea útil y referenciado a lo que nos queremos dirigir.

Diseño conceptual:

En este apartado debemos definir sin lujo de detalle el funcionamiento del sitio, su navegación y la estructura con la que se va a contar. La arquitectura de la pagina web se debe mostrar a través de gráficos que ilustren con detalle lo que se quiere proyectar, para así tener una idea clara en el desarrollo del funcionamiento de la página o sitio web.

Diseño visual:

En base al diseño conceptual que se desarrolló, se puede hacer uso mayor resolución y detalles y la idea que se tiene, aquí se tienen que definir con estructura y detalle el resultado al que se quiere llegar. Es importante que tenga un estilo que sea apto para los usuarios, que sea entendible y que presente la información de forma detallada y resumida.
